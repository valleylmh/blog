# 同一个提示词跑了几个大模型后，我第一次真切感受到差距

今晚，我突然有点想把一件小事写下来。

起因其实很简单。我拿了同一个提示词，让几个不同的大模型去完成同一个任务：先阅读 Claude Code 里关于 Agent Teams 的文档，再用 SVG 画出一张带动态效果的架构图，而且 UI 还要尽量符合现代设计的最佳实践。



**这不是一道特别标准化的题，但正因为它不标准，反而很适合拿来验证模型能力。它既考验文档理解能力，也考验信息提炼能力；既要有结构化表达，又要能真正生成一个能交付的 SVG 文件；再往前一步，还带上了动态效果和审美要求。换句话说，这不是“会不会答题”，而是“能不能把事情做成”。**



我目前试了几个模型：openai-codex/chatgpt-5.4、qwen3.5-plus，还有中转站里大家常说的 claude code opus 4.6。如果只说“都能做”，那其实不够准确。更真实的说法是：它们都能开始做，但最后做出来的完成度、稳定性、审美和细节处理，差别还是挺明显的。

这次给我最好印象的，确实是 opus 4.6。

它的“好”不只是结果更漂亮，也不只是代码更完整，而是一种整体上的成熟感。你会觉得它更理解这个任务到底要什么：不是简单拼一个架构示意图，而是要把“文档理解”“信息抽象”“视觉表达”“动态呈现”这些层面揉到一起。很多模型也能生成一张图，但更像是在“完成要求”；而 opus 4.6 做出来的东西，会更像一个有经验的协作者在认真交付作品。

这种差别其实挺有意思的。平时大家聊模型强弱，很容易落到参数、榜单、跑分，或者一句模糊的“这个更聪明、那个更强”。可一旦把它们放进真实任务里，差异就不再抽象了。它会直接体现在作品里：结构是不是清晰，细节是不是稳，动效是不是自然，视觉是不是有节制，甚至连“有没有现代感”这种原本很主观的东西，最后也会变得有体感。



我越来越觉得，验证大模型能力，最好的方式不是问它会不会，而是给它一个稍微完整一点的任务，看它最终能交付出什么。因为真正的能力差异，从来都不只在回答里，而是在作品里
